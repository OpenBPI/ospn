# 开放式社交公共网络  
开放式社交公共网络（英文为Open social public network，简称OSPN）是一套能够让所有应用（APP）产生“跨界交流”的解决方案。

## 理念  
&emsp;&emsp;开放式社交公共网络的理念是打破社交垄断壁垒，帮助企业留住用户，帮助用户保护隐私。本着“开放”、“公共”的原则，OSPN采用了完全去中心化的设计，定位是“公共网络”，对于每一个企业而言都可以自由接入任意使用。

&emsp;&emsp;我们认为社交垄断对社会、企业、用户都会造成潜在的威胁。因此我们希望用户能够分散，使用多家公司的产品，而又保持应有的联系。

&emsp;&emsp;对于企业而言，一个产品的用户流失，一方面是因为产品新鲜感已经过去，另一个方面是因为产品没有建立起足够的用户依赖。社交是产品建立用户依赖最有效的手段，因为新生代每年都会产生，因此随便什么时候进入社交领域都不算晚。

&emsp;&emsp;对于用户而言，隐私保护非常重要，而企业肯定是会想办法收集用户隐私并且用作数据分析，一个会收集你隐私进行利用的企业又怎么会全心全意帮助用户保护隐私呢？这明显就是一个悖论。因此我们提出了“宏观隐私保护”、“微观隐私保护”、以及“反大数据”的理念。

### 宏观隐私保护
&emsp;&emsp;社交垄断意味着我们所有的隐私都会集中在一家企业。包括我们的个人隐私资料和我们的朋友关系网，这对于你和你的朋友都是一种不好的状态。宏观隐私保护的理念是你和你的朋友们使用多个企业的产品，这样你们的隐私资料就会分散到多家企业，而企业只能得到你的一部分关系网数据，做大数据分析的准确率就会降低。而如果你是一个特别注重隐私保护的人，选择一家收费的企业，更有利于保护你的隐私。

### 微观隐私保护
&emsp;&emsp;人类的私密聊天特点是，我告诉了你一句话，当你把这句话告诉第三人“是我说的”的时候，这个第三人是无法判断这句话是否是我说的。  
&emsp;&emsp;我们的主张是在网络上的私密聊天也要达到这样的效果，私密聊天除了你们两人，因为无法保证聊天记录不被传出（可能是你，可能是他，即使是语言聊天也无法保证不传出），但是我们希望这个记录无法被作为证据。这才符合私密聊天的特性。  
&emsp;&emsp;而目前的聊天软件是无法做到这一点的，比如微信的聊天记录可以作为证据。阅后即焚也不是最好的设计，因为你必须拉着你的朋友使用同一款应用，而如果他不同意使用，则无法做到。  
我们的微观隐私保护可以做到这一点，请参见后面我们更新的解决方案。

### 反大数据
&emsp;&emsp;大数据分析会精准定位一个人的身份，消费水平等等，会将一个人进行归类。企业收集了我们的隐私，我们并不知道他们会做怎样的分析，而这一块是没有监管的，我们不希望看到企业分析我们的数据。我们无法做到反对企业分析数据，只能尽量通过数据分散化来加大对大数据分析的难度。

## 接入OSPN网络
&emsp;&emsp;接入OSPN网络需要企业对现有系统进行一定的适配，包括客户端和服务器的适配。服务器的适配需要企业搭建osn-connector，请参见开源项目[osn-connector](https://github.com/OpenBPI/osn-connector)。
企业IM服务与osn-connector之间的数据交换请参见例子[osn-imsdemo](https://github.com/OpenBPI/osn-imsdemo)
